Business Requirement Document (BRD)
Core Banking Operations Portal (CBOP)
1. Project Overview
1.1 Purpose

The Core Banking Operations Portal (CBOP) is an internal web-based application designed to support branch-level banking operations. The system enables authorized bank staff to manage customers, accounts, transactions, approvals, audits, and reports in a secure, role-based, and auditable environment.

This portal is not customer-facing and will be used exclusively by internal stakeholders such as:

Branch employees

Branch managers

Auditors

System administrators

1.2 Business Objectives

Digitize manual branch banking operations

Enforce strict role-based access control

Implement maker–checker (approval) workflows

Maintain complete and immutable audit trails

Generate operational and regulatory reports

Secure sensitive customer and transaction data

1.3 Target Users
Role	Description
Teller	Performs day-to-day customer and transaction operations
Branch Manager	Approves high-value transactions
Auditor	Reviews audit logs and reports
System Administrator	Manages users, roles, and system configuration
